# <a name="map-a-sharepoint-library-to-a-network-drive"></a>Mapear uma biblioteca do SharePoint para uma unidade de rede

O mapeamento de uma biblioteca como uma unidade de rede é temporária e suportadas apenas através do Internet Explorer. Ocasionalmente, tem de abrir o site SharePoint no Internet Explorer e seleccionar a **manter-se sessão** para impedir que a sessão expirar. Em vez disso, [sincronizar ficheiros do SharePoint com o novo cliente de sincronização de OneDrive](https://support.office.com/article/sync-sharepoint-files-with-the-new-onedrive-sync-client-6de9ede8-5b6e-4503-80b2-6190f3354a88) </a> que fornece [Ficheiros a pedido](https://support.office.com/article/learn-about-onedrive-files-on-demand-0e6860d3-d9f3-4971-b321-7092438fb38e). Aceder a todos os ficheiros de OneDrive sem utilizar o espaço de armazenamento local.

Se optar por mapear uma unidade em vez de [utilizar o novo cliente de sincronização de OneDrive](https://support.office.com/article/sync-sharepoint-files-with-the-new-onedrive-sync-client-6de9ede8-5b6e-4503-80b2-6190f3354a88), certifique-se de que siga os passos do artigo abaixo. 


**Como configurar e resolver problemas de unidades de rede mapeadas**


Consulte a [resolução de problemas computadorquot que ligar a Onlines do SharePoint](https://docs.microsoft.com/sharepoint/support/administration/troubleshoot-mapped-network-drives).

Nota: Se utilizar o Internet Explorer 10 com Windows 8 ou o Windows 7 e receber **acesso negado** ou **o caminho é não está acessível** quando mapear uma unidade, instale [Esta correcção](https://support.microsoft.com/help/2846960) para resolver este problema.
