# <a name="map-a-sharepoint-library-to-a-network-drive"></a>Mapeie uma biblioteca sharepoint para uma unidade de rede

Mapear uma biblioteca como uma unidade de rede é temporário e suportado apenas através do Internet Explorer. Você deve ocasionalmente abrir o site SharePoint no Internet Explorer e selecionar **Stay assinado para** evitar que a sessão expire. Em vez disso, [sincronizar arquivos SharePoint com o novo cliente](https://support.office.com/article/sync-sharepoint-files-with-the-new-onedrive-sync-client-6de9ede8-5b6e-4503-80b2-6190f3354a88) </a> de sincronização OneDrive que fornece [arquivos demanda.](https://support.office.com/article/learn-about-onedrive-files-on-demand-0e6860d3-d9f3-4971-b321-7092438fb38e) Acesse todos os seus arquivos no OneDrive sem usar espaço de armazenamento local.

Se você optar por mapear uma unidade em vez de usar o novo cliente de [sincronização OneDrive,](https://support.office.com/article/sync-sharepoint-files-with-the-new-onedrive-sync-client-6de9ede8-5b6e-4503-80b2-6190f3354a88)certifique-se de seguir as etapas do artigo abaixo. 


**Como configurar e solucionar solucionados unidades de rede**


Veja [as unidades de rede mapeadas troubleshoot que se conectam ao SharePoint Onlines.](https://docs.microsoft.com/sharepoint/support/administration/troubleshoot-mapped-network-drives)

NOTA: Se você usar o Internet Explorer 10 com Windows 8 ou Windows 7, e receber **acesso negado** ou path não **é acessível** ao mapear uma unidade, instale [esse hotfix](https://support.microsoft.com/help/2846960) para resolver esse problema.
