# <a name="workflows-in-sharepoint"></a>Fluxos de trabalho no SharePoint

Fluxos de trabalho no SharePoint permitem modelar e automatizar processos de negócios. O SharePoint suporta dois ambientes primários de desenvolvimento de fluxo de trabalho para a autoria de fluxos de trabalho: SharePoint Designer e Visual Studio. 

Veja desenvolver fluxos de [trabalho sharepoint em SharePoint Designer e Visio](https://docs.microsoft.com/sharepoint/dev/general-development/develop-sharepoint-workflows-using-visual-studio) ou desenvolver fluxos de [trabalho SharePoint usando visual studio](https://docs.microsoft.com/sharepoint/dev/general-development/develop-sharepoint-workflows-using-visual-studio) para avaliar as vantagens e desvantagens de cada um. 

Se você está tentando determinar se os fluxos de trabalho são uma solução para usar em seu ambiente SharePoint ou estão procurando informações gerais sobre como começar, veja a [visão geral dos fluxos de trabalho no SharePoint](https://docs.microsoft.com/sharepoint/dev/general-development/get-started-with-workflows-in-sharepoint#overview-of-workflows-in-sharepoint).
